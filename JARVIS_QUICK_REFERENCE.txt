JARVIS is triggered by any change in your workspace:

Key Capabilities:
1. MCP Integration
   → Polls OpenCode conversations every 60 seconds
2. File System Monitoring
   → Instant trigger when `notes.json` or `deals/` change
3. Modular Skill Execution
   → Each document (Risk, MEDDPICC, etc.) is a separate "skill"
4. Cross-Skill Data Aggregation
   → Skills read each other's outputs for maximum context
5. Dashboard Synthesis
   → Every account gets a premium, master DASHBOARD.html

Result: JARVIS provides real-time account intelligence.

🔗 HOW IT ALL CONNECTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Event-Driven Architecture:
  Observers → Events → Orchestrator → Skills → Files → Dashboard

Key Flow:
  1. Observer sees conversation update → publishes EVENT
  2. Orchestrator triggers relevant SKILLS
  3. Skills gather context (notes + other skills) → generate MD
  4. Dashboard skill aggregates all MD → generates HTML
  5. User views premium intelligence in DASHBOARD.html

Full details: docs/architecture/DATA_FLOW.md
